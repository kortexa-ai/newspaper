GitHub has shipped a `/security-review` slash command into the Copilot
app, now in public preview, that scans a developer's current
work-in-progress code for vulnerabilities before it's ever committed
([GitHub Changelog](https://github.blog/changelog/2026-07-14-security-reviews-now-available-in-the-github-copilot-app/)).

Run the command, and it analyzes the changes in your current workstream
and returns findings scored by severity and confidence, with concrete,
actionable suggestions attached. It's built to catch the vulnerability
classes that show up constantly in real-world code review — injection
flaws, cross-site scripting, insecure data handling, path traversal,
weak cryptography — without requiring a separate security tool or a
trip out of the editor.

The notable part isn't the feature itself, which extends scanning
GitHub already offered in its Copilot CLI into the everyday coding app
developers use most. It's who gets it: the command is available to
Copilot Free, Pro, Business, and Enterprise users alike during the
preview, not gated behind an enterprise contract. Security review has
historically been something teams paid extra for, layered on top of
regular coding tools as a separate product. Folding a version of it into
the free tier of the most widely used AI coding assistant means a much
larger set of hobbyists, students, and small teams get a basic
vulnerability check for the first time — the kind of unglamorous,
broadly distributed improvement that quietly raises the floor for
everyone, rather than the ceiling for a few.
